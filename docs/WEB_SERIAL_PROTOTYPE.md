# TN-GridSense Web Serial Hardware Prototype Bridge

## Overview
The Live Hardware Prototype feature enables direct, zero-configuration communication between the TN-GridSense cloud dashboard running in the browser and the physical ESP32 Smart Pole node via a USB connection. 

By leveraging the modern Web Serial application programming interface, the browser itself acts as a local gateway. It reads the raw telemetry data generated by the ESP32 firmware, parses it locally, and injects it directly into the full-scale Next.js and Elysia cloud architecture.

This bridges the gap between hardware prototyping and cloud deployment without requiring intermediate MQTT brokers, Python bridging scripts, or physical Wi-Fi routing during the testing phase.

---

## Architecture Flow

The data travels through a seamless end-to-end pipeline:

1. The Physical ESP32 Node sends space-delimited text over the USB serial connection.
2. The Client Browser captures this stream via the Web Serial decoder.
3. A local parser cleans the data and formats it into a structured telemetry object.
4. The Browser sends an HTTP POST request to the Elysia Backend API.
5. The Backend updates its in-memory state and broadcasts the new data to the Next.js Dashboard over a WebSocket channel.
6. The Dashboard receives the signal and instantly updates the live graphical interface without reloading the page.

---

## Process Breakdown

### 1. The Hardware Layer 
The ESP32 microcontroller runs the custom grid firmware. In its main loop, it reads sensors for Voltage, AC Current, DC Current, and Temperature. It formats this data into a simple, human-readable text line and pushes it over the standard USB Serial interface for debugging.

### 2. The Browser Gateway Layer
When a user navigates to the Prototype page in a supported browser like Chrome or Edge, they are presented with a hardware bridge interface.

The application automatically detects and reconnects to a previously authorized ESP32 without prompting the user. If it is the first time connecting, the user clicks to connect, which triggers the browser's native secure USB prompt.

The connection is opened at a baud rate of 115200 to match the configuration baked into the ESP32 code.

### 3. Stream Processing & Parsing
The raw serial byte stream is ingested continuously. Because serial data chunks arrive inconsistently over USB, the logic safely buffers the incoming chunks and splits them line by line.

Once a complete line is extracted, it runs through a text parser to pull out the precise numerical values for voltage, current, power, temperature, and current safety status.

### 4. Cloud Integration 
The parsed metrics are packed into a standard telemetry data payload.

Crucially, the Web Serial client assigns the physical board a Virtual Identity. It tags the board as a specific prototype pole located in the Chennai district, tied to a prototype feeder line.

It then forwards this data to the Elysia backend. The backend processes this exactly as if it arrived from an in-field cellular node:
First, it updates the central data store. Next, it triggers anomaly and predictive maintenance analytics routines. Finally, it broadcasts the update to all actively connected clients monitoring the Chennai district via the live WebSocket channel.

### 5. Data Visualization 
Simultaneously, the Prototype page provides immediate visual feedback. 

Large, color-coded dashboard components update instantly to show Volt, Amp, Temp, and Status. A toggleable raw serial monitor log is also available, displaying the raw text stream as it arrives. This log automatically scrolls to the newest line, allowing developers to debug the hardware output exactly as they would in a traditional hardware monitor tool.

---

## Strategic Advantages

First, it creates an immediate feedback loop. Hardware engineers can tweak sensor calibration on the device and instantly see the results visualize on a state-scale cloud dashboard without deploying any networking code.

Second, it allows true system end-to-end testing. The serial bridge tests the entire software stack including parsers, the backend API, the analytics engine, the data store, and the front-end rendering engines. It is an exact replica of production, simply substituting the physical transmission medium.

Finally, it grants premium demonstration capabilities. It permits live, impressive "Hardware-in-the-Loop" demonstrations anywhere using just a laptop and a USB cable, which is critical for hackathons and stakeholder presentations where networking might be unreliable.
